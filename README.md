Rationale
Business Context
Greenfield Local Hub (GLH) is a Local cooperative of farmers and food producers which would like to become digital. GLH already offers a single retail outlet, a collective marketing for local farmers and food producers, product management, stock control and product traceability.
They would like to have a digital solution to help inform customers about their producers, methods and the benefits of buying locally; they would like to allow for customers to place orders for collection or delivery and manage their accounts. In the digital solution I am proposing I would like to develop all the features that the client requires and some of the features that were suggested by the client after having conducted market research with existing customers.
Functional and Non-Functional Requirements
Functional Requirements
Homepage
Needs to give a general overview of GLH.
Show pictures and promotional offers. 
Navigation bar
This should contain the links to all the different pages. 
Producers Page 
This page should list all the farmers and the food producers working with GLH. 
It should show details such as the name, farm location, farming methods, product types. It will also allow the customers to view the products that the producers are selling. 
Products Page 
This will display all the available products. 
It will allow the customers to see product details including the description, the price, the producer, stock availability. 
Customer Accounts 
Customers should be able to create an account, sign in and sign out, change/update their personal details, see their order history and manage delivery or collection preferences. 
Order placement 
Customers should be able to add products to their shopping cart, place orders, choose a method of receiving the order – Either collection or delivery -, receive an order confirmation. 
Order management 
The customer can see their order status (pending, out for delivery, completed). They also will receive emails notifying them of their order status changes. 
They can choose a preferred time slot for collection. 
Loyalty Scheme 
The customers will earn points for each purchase – a set amount per pound spent. These points can be redeemed for discounts on orders. 
The customers will be able to view their points balance in their account. 
Producer dashboard 
The producers should be able to log into their own dashboard, add and update product information. 
It should allow for the producer to update stock levels, view incoming orders and see what products are selling. 
Stock control 
The stock levels should be automatically changed whenever an order is placed. Producers can manually change/update stock. 
The out-of-stock products won’t be able to be ordered. 
Product Traceability 
The products will each show the producer, farm location and production method.
Non-Functional Requirements 
Usability 
The website needs to be easy to use, and it should be easy to navigate. It needs to have a clear and simple layout. 
Performance 
The pages need to have a quick loading time – roughly 2-3 seconds. It needs to be able to support multiple orders being placed at the same time. 
Security 
The accounts need to have secure passwords. 
Requires validation. 
Reliability 
Data must be stored safely. 
The system needs to be able to be available 24/7 without crashing.
Decomposition
Functional Requirements decomposition
Homepage
Must provide a homepage which will introduce you to GLH and will inform the customers about the producers and products. Make sure information is presented clearly.
Producers Page
This page must display information about farmers and producers. The data must be organized. To do this, the data will be structured so customers can easily browse.
Products Page
Allow for customers to browse different products and see detailed information about it before buying. Need to be able to filter, see product prices, stock.
Customer accounts
Customers need to be able to create accounts and manage their information. It needs to have a secure system where they can register log in and view data about their account such as order history.
Order Placement
Customers need to be able to easily place orders. Needs a shopping cart where customers can add or remove products before buying, select delivery or collection and confirm the order.
Order Management
Customers need a way to manage and track their orders. It needs a system that shows status and send confirmation emails for changes.
Loyalty Scheme
The system needs to reward customers with loyalty points. Create a system which adds a certain amount of points based on their purchase amount. These can be used later for discounts.
Producer Dashboard
Producers must be able to manage their products and stock. Create a separate secure dashboard for producers where accounts can only be made by contacting the GLH. It needs to be able to add products, update details, adjust prices, adjust stock levels, and view incoming orders.
Stock control
The information about stock needs to be accurate to prevent mishaps where customers order unavailable products that are out of stock. They need to link the stock levels to the ordering system so it automatically updates after purchases. If there isn’t enough stock, block purchases.
Product Traceability
Customers need to know where their food comes from. The products will be linked to the relevant producer and contain necessary information.
Non-Functional Requirements decomposition
Usability
Customers need a simple to navigate website. Create a simple user interface with clear menus, structuring and readable text.
Performance
Need to have fast loading times since slow loading pages reduce customer satisfaction. Need to optimise by using efficient queries in the database, image compression and hosting on reliable servers.
Security
Accounts and personal information must be protected. Implementation of secure password storage, validation and user authentication combined with data encryption and secure login systems will ensure data remains secure.
Reliability
The system needs to remain always available and prevent any sort of data loss. Use of reliable hosting and regular backups ensure the system is constantly online and protects data.
Key performance indicators 
Customer satisfaction 
This is to measure the users experience with the platform and we can collect this through customer ratings or feedback. 
It will indicate whether the website is easy to use and if it is meeting the customer needs or not. 
Revenue growth 
This is to see whether the website is having a positive effect on the sales. We can see this through a percent increase in sales. It will show whether the digital solution is helping the GLH. 
Stock system accuracy 
This is to ensure that all information about the stock is reliable and accurate. A high accuracy would mean that the system is working well. This will prevent customers from ordering products that have none or not enough stock. 
System performance 
This is to measure the websites technical performance. An example is the page loading time or whether the website can handle multiple users and orders. An efficient website increases the user experience and helps customer retention. 
Description of proposed solution
In the content above, I decomposed, analysed and set the requirements for the solution. Below is going to be a summary of the proposal and the main benefits to the client if they implement it. The proposed solution will allow for customers to view information about the local farmers and the food producers, look at available products and stock, place online orders for collection or delivery. 
The system will contain features such as customer accounts, order management, a collection of products, and a loyalty point system aimed at encouraging retention and repeated orders. For the producers and farmers, the platform will contain a producer platform where they can manage their products, manage stock levels, and view orders.
